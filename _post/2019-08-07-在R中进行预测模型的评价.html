在R中进行预测模型的评价

当我们进行了模型的建立之后，需要对模型进行评价。这部分在论文中也是主要的部分之一。R作为一个完善和强大的统计软件，对于所有的模型都有一个较为统一的评价方法。

在这里，我们使用《R语言实践》第17章中所展示的内容进行logistic回归模型和随机森林的模型建立，并且通过不同的方法来比较模型的优异。至于随机森林和决策树，我将会在以后的内容中进行介绍，

数据集准备

我们这里使用的是威斯康星州乳腺癌数据集作为我们的训练和验证数据集。这部分的内容可以在《R语言实践》中找到。

首先我们需要下载这个数据集，并且将数据集随机分为训练集和验证集：

loc <- "http://archive.ics.uci.edu/ml/machine-learning-databases/"
ds <- "breast-cancer-wisconsin/breast-cancer-wisconsin.data"
url <- paste(loc, ds, sep = "")
breast <- read.table(url, sep = ",", header = FALSE, na.strings = "?") 
names(breast) <- c("ID", "clumpThickness", "sizeUniformity", 
                   "shapeUniformity", "maginalAdhesion",
                   "singleEpithelialCellSize", "bareNuclei", 
                   "blandChromatin", "normalNucleoli", "mitosis", "class")
df <- breast[-1] 
df$class <- factor(df$class, levels = c(2, 4), labels = c("benign", "malignant"))
set.seed(1234) 
train <- sample(nrow(df), 0.7 * nrow(df)) 
df.train <- df[train, ] 
df.validate <- df[-train, ] 
print(table(df.train$class))
## 
##    benign malignant 
##       329       160
print(table(df.validate$class))
## 
##    benign malignant 
##       129        81

我们打印出两个数据集中恶性乳腺癌和良性乳腺癌的比例是否接近一致。如果存在很大程度的差异，我们可能要在set.seed()中重新选择一个随机种子数来进行抽样分组。

当然，可能使用百分比会更加直观一些：

print(prop.table(table(df.train$class)))
## 
##    benign malignant 
## 0.6728016 0.3271984
print(prop.table(table(df.validate$class)))

## 0.6142857 0.3857143

模型的建立

我们在这里建立logistic回归模型和随机森林模型来预测良性和恶性乳腺癌。

logistic 回归模型

logistic 回归是一种在医学领域中被广泛运用的模型，除了用来作为分类器以外，还可以用来寻找危险因素（使用OR来进行表示）。在R中，logistic 回归的模拟拟合是内置在basic中的，我们可以直接使用glm()进行拟合：

随机森林

随机森林是一种使用多个机器学习进行拟合的方法，他是决策树的一种特有方式。除此之外还有bagging和Boosting，这两种方法并不是一个机器学习算法特有的。具体的内容将会在以后讲决策树的模型建立中具体去讲。这里，我们仅仅去选择随机森林来建立模型。随机森林的函数我们可以在randomForest包中找到。如果你需要进行这一步，你需要确保自己的机器已经安装了这个包。如果没有安装，请使用install.packages("randomForest")进行安装。

library(randomForest)
## randomForest 4.6-14
## Type rfNews() to see new features/changes/bug fixes.
set.seed(1234)
fit.forest <- randomForest(class ~ ., data = df.train, 
                           na.action = na.roughfix, importance = TRUE)
print(fit.forest)

模型的评价

这样，我们就使用训练集建立了我们想要的分类器。现在我们需要用这个建好的分类器去验证集中验证和评价模型是否能够很好的把区分恶性乳腺癌和良性乳腺癌。

模型预测

在R中，我们可以使用predict()函数进行模型的预测。

logit.pred <- predict(fit.logit, df.validate, type = "response")
forest.pred <- predict(fit.forest, df.validate, type = "class")
print(head(logit.pred))
##           3           4           5           8          14          18 
## 0.006402885 0.841424146 0.013774681 0.004145666 0.003148881 0.007721851
print(head(forest.pred))
##         3         4         5         8        14        18 
##    benign malignant    benign    benign    benign    benign 
## Levels: benign malignant

logistic 回归模型的预测类型因为预测的是概率，所以我们需要选择response，而随机森林本来就是二分类器，所以我们选择了class，这里的type还可以传递prob，我们将在ROC曲线的绘制中，使用这个参数进行ROC的绘制。

混淆矩阵

混淆矩阵是一种我们非常常用的方法来衡量模型的优异程度。在医学领域中经常听到的灵敏度和特异度便是由混淆矩阵计算得来的。关于混淆矩阵（confusion matrix）的更多内容，可以参看wiki百科的内容，以及网站上优秀的博文。由混淆矩阵我们计算很多评价模型的参数，如灵敏度、特异度、准确性等等。我们在这里，会使用caret包中的confusionMatrix()函数来获得混淆矩阵。

ROC曲线

或许前面繁多的统计量在论文中不怎么见到，而ROC曲线在各个期刊论文中都可以看到，毕竟谁都喜欢可视化的图不是么？一般来讲，能够使用predict()函数进行预测的，我们都可以将其绘制成ROC曲线。在这里我们需要使用的包是ROCR，这个包提供了ROC的计算和AUC面积的计算。如果要在本地机器中实现接下来的步骤，请确保你的机器中已经安装了这个包，如果没有，请使用install.packages("ROCR")来进行安装。

library(ROCR)
## Loading required package: gplots
## 
## Attaching package: 'gplots'
## The following object is masked from 'package:stats':
## 
##     lowess
forest.prob <- predict(fit.forest, df.validate, type = "prob")
forest.predt <- prediction(predictions = forest.prob[ , 2],
                   labels = df.validate$class)
forest.perf <- performance(forest.predt, measure = "tpr", x.measure = "fpr")
plot(forest.perf)
abline(a = 0, b = 1)

print(round(performance(forest.predt, "auc")@y.values[[1]], 4))
## [1] 0.9945

这样，我们就绘制出了ROC曲线，在performance()函数中，我们传递了measure = "tpr"，即真阳性率（True positive rate）作为Y轴，使用假阳性率（False positive rate）作为X轴。这种表现方式在医学领域中较为少见，我们通常使用灵敏度和1-特异度来作为ROC曲线的Y轴和X轴。在performace()函数中，提供了measure = "sens"和x.measure = "spec"来计算灵敏度和特异度。

曲线下面积的计算也是通过performace()函数进行的，我们可以使用performance(forest.predt, "auc")@y.values[[1]]提取出AUC的值。

forest.perf <- performance(forest.predt, measure = "sens", x.measure = "spec")
plot(forest.perf)

这时，我们可以看到它绘制了一个反过来的曲线，这是因为我们使用的是特异度而不是1-特异度。

同时，我们或许还需要将logistic回归的ROC曲线和随机森林的ROC曲线放到一块儿进行对比，然后还要在图中标记两种模型的AUC面积。这时，我们可能需要一点小小的创新来实现我们的目标。

这样，我们就看到了我们想要的信息：

  • S4 type object：这是R标准的面向对象实现方式。emmmm……，这么讲不容易懂，这里只要知道的是，和list不同，提取出S4类型的数据我们不再使用$而是使用@。
  • x.name：Specificity，这个项目的值就是我们的X轴的值。什么？你说X轴是1-specificity？额，提取出来自己减一下不就完了？
  • y.name: Sensitivity，这和上边的一样，但是他们俩都是一个字符串，我们真正需要的值在x.values和y.values中。

这样，我们思路明确了，将performace()函数输出的x.values和y.values提取出来，放到我们自己创建的数据框中，然后使用ggplot进行绘制ROC曲线。这样，我们就可以为所欲为啦。

当然，我们最终的目标是要把两个ROC曲线放一块儿的。这是我们需要额外的一列来指明x.values和y.values是哪个模型中的。接下来看代码吧：

library(ROCR)
forest.prob <- predict(fit.forest, df.validate, type = "prob")
forest.predt <- prediction(predictions = forest.prob[ , 2],
                   labels = df.validate$class)
forest.perf <- performance(forest.predt, measure = "sens", x.measure = "spec")
logit.predt <- prediction(logit.pred, df.validate$class)
logit.perf <- performance(logit.predt, measure = "sens", x.measure = "spec")
df.forest <- data.frame("model" = "randomforest",
                        "Specificity" = 1 - forest.perf@x.values[[1]],
                        "Sensitivity" = forest.perf@y.values[[1]])

df.logit <- data.frame("model" = "logistic",
                       "Specificity" = 1 - logit.perf@x.values[[1]],
                       "Sensitivity" = logit.perf@y.values[[1]])
df.roc <- rbind(df.forest, df.logit)
logit.auc <- round(performance(logit.predt, "auc")@y.values[[1]], 4)
forest.auc <- round(performance(forest.predt, "auc")@y.values[[1]], 4)
print(df.roc[c(1:3, 73:75), ])
